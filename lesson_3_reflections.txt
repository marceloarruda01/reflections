When would you want to use a remote repository rather than keeping all you work local?
  
  Quando quiser compartilhar o trabalho com outras pessoas, quando quiser receber 
  contribuições de outras pessoas ou quando quiser acessar o repositório a partir de 
  outros computadores.

Why might you want to always pull changes manually rather than having Git automatically
stay up-to-date with your remote repository?
	
	Porque o Git pode sobrescrever algumas alterações que estou fazendo localmente no 
	mesmo arquivo na minha working area caso não tenha criado um branch para isso.

Describe the differences between forks, clones, and branches. When would you use one 
instead of another?

	Forks são cópias, ou clones, de repositórios que são realizados dentro da ferramenta 
	GitHub. São utilizados para criar cópias de repositórios no GitHub que serão
	modificados sem alterar o repositório de origem.

	Clones são cópias de repositórios que podem ser feitas localmente entre pastas do 
	mesmo computador, ou remotamente a partir de um repositório online para o seu
	computador. São utilizados para trabalhar nos arquivos do repositório de forma colaborativa com outros membros do projeto.

	Branches são caminhos, ou ramos, distintos de trabalho dentro do mesmo repositório 
	que contém a história de um ou mais commits. São utilizados para testes, correções e/ou criações de novas funcionalidades.

What is the benefit of having a copy of the last known state of the remote stored locally?

	Você pode comparar o trabalho no seu branch com as modificações mais recentes que 
	foram realizadas no servidor, inclusive podendo ir fazendo o merge delas para 
	incorporá-las no seu brach.
